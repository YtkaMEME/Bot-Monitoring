# Бот для обработки данных анкетирования

Телеграм-бот для обработки и анализа данных из анкет в формате Excel.

## Возможности

- Загрузка Excel файлов с данными анкет
- Получение отчета из Anketolog по названию анкеты
- Автоматический анализ различных типов вопросов:
- Поддержка NPS и CSI вопросов
- Сохранение результатов в Excel и CSV
- Опциональная загрузка обработанного Excel файла на Яндекс.Диск с отправкой ссылки пользователю
- Административные функции для управления списками пользователей и мусорных слов

## Установка

1. Клонировать репозиторий:
   ```bash
   git clone <ссылка на репозиторий>
   ```

2. Установить зависимости:
   ```bash
   pip install -r requirements.txt
   ```

3. Настроить конфигурацию:
   - Скопировать `.env.example` в `.env`
   - Заполнить в `.env`:
     - `ANKETOLOG_TOKEN`
     - `YANDEX_DISK_TOKEN`
     - `YANDEX_REPORTS_FOLDER`
   - Создать локальный `TOKEN.py` на основе `TOKEN.example.py`
   - При необходимости создать локальный `PROXY.py` на основе `PROXY.example.py`

4. Создать необходимые директории:
   ```bash
   mkdir -p downloads logs
   ```

## Запуск

```bash
python main.py
```

## Использование

1. Отправить боту Excel файл с данными анкеты
2. Или нажать кнопку `Скачать отчет из Anketolog` и указать название анкеты
3. Выбрать тип обработки и дополнительные параметры
4. Получить обработанные данные в форматах Excel и CSV
5. При необходимости загрузить обработанный Excel файл на Яндекс.Диск

## Команды

- `/start` - Начать работу с ботом
- `/cancel` - Отменить текущую операцию
- `/get_my_id` - Получить свой Telegram ID
- `/change_del_list` - Управление списком мусорных слов (требуются права)
- `/admin` - Административная панель (требуются права администратора)

## Конфигурация

- `TOKEN.py`:
  локальный файл с Telegram Bot token, не должен попадать в git
- `PROXY.py`:
  локальный файл с proxy, если нужен для запуска
- `.env`:
  локальный файл с токенами Anketolog и Яндекс.Диска
- `config/allowed_users.json`, `config/admins.json`, `config/list_to_del.json`:
  создаются приложением автоматически при первом запуске, если отсутствуют

## Требования

- Python 3.8+
- aiogram 3.7.0+
- pandas 2.2.2+
- openpyxl 3.1.4+ 

## Публикация на GitHub

- В репозиторий не должны попадать:
  - `.env`
  - `TOKEN.py`
  - `PROXY.py`
  - временные файлы из `downloads/`
  - отладочные скрипты вроде `test_download.py` и `file_list.py`
- Перед публикацией убедитесь, что в коммитах и diff нет реальных токенов и proxy-данных.
